Цель работы

Изучить влияние числа потоков на производительность вычислений. Создать массив большого размера, реализовать вычисление суммы элементов массива в однопоточном режиме, с использованием 2 потоков и с использованием 4 потоков. Измерить время выполнения и сравнить результаты.

Теоретическая справка

Многопоточность позволяет выполнять несколько потоков параллельно. Если задачу можно разделить на независимые части, каждый поток обрабатывает свою часть данных, что сокращает общее время выполнения.

При суммировании массива каждый поток считает сумму своего диапазона элементов. После завершения работы всех потоков частичные суммы складываются.

Особенность Python (GIL)

В CPython есть GIL (Global Interpreter Lock). Из-за него модуль threading не ускоряет вычислительные задачи. Для настоящего параллелизма в Python нужен модуль multiprocessing. В C++ потоки std::thread работают по-настоящему параллельно.

Описание программы

  1. Создается массив из 10 000 000 случайных чисел от 1 до 100.
  2. В однопоточном режиме сумма считается одним циклом.
  3. В многопоточном режиме массив делится на равные части по числу потоков.
  4. Каждый поток считает сумму своей части.
  5. Частичные суммы складываются в итоговый результат.
  6. Для каждого варианта измеряется время выполнения.

Исходный код

import threading
import time
import random

ARRAY_SIZE = 10_000_000
array = [random.randint(1, 100) for _ in range(ARRAY_SIZE)]

def sum_part(arr, start, end, results, index):
    s = 0
    for i in range(start, end):
        s += arr[i]
    results[index] = s

def single_thread_sum(arr):
    s = 0
    for v in arr:
        s += v
    return s

def multi_thread_sum(arr, num_threads):
    threads = []
    results = [0] * num_threads
    chunk = len(arr) // num_threads
    for i in range(num_threads):
        start = i * chunk
        end = start + chunk if i < num_threads - 1 else len(arr)
        t = threading.Thread(target=sum_part, args=(arr, start, end, results, i))
        threads.append(t)
    for t in threads:
        t.start()
    for t in threads:
        t.join()
    return sum(results)

start = time.time()
s1 = single_thread_sum(array)
t1 = time.time() - start
print(f"1 поток: сумма={s1}, время={t1:.4f} сек")

start = time.time()
s2 = multi_thread_sum(array, 2)
t2 = time.time() - start
print(f"2 потока: сумма={s2}, время={t2:.4f} сек")

start = time.time()
s4 = multi_thread_sum(array, 4)
t4 = time.time() - start
print(f"4 потока: сумма={s4}, время={t4:.4f} сек")
#include <iostream>
#include <vector>
#include <thread>
#include <chrono>
#include <random>

const int SIZE = 10000000;

void partial_sum(const std::vector<int>& arr, int start, int end, long long& result) {
    long long s = 0;
    for (int i = start; i < end; i++) s += arr[i];
    result = s;
}

long long single_sum(const std::vector<int>& arr) {
    long long s = 0;
    for (int v : arr) s += v;
    return s;
}

long long multi_sum(const std::vector<int>& arr, int n) {
    std::vector<std::thread> threads;
    std::vector<long long> results(n, 0);
    int chunk = arr.size() / n;
    for (int i = 0; i < n; i++) {
        int s = i * chunk;
        int e = (i == n - 1) ? arr.size() : s + chunk;
        threads.emplace_back(partial_sum, std::ref(arr), s, e, std::ref(results[i]));
    }
    for (auto& t : threads) t.join();
    long long total = 0;
    for (long long r : results) total += r;
    return total;
}

int main() {
    std::vector<int> arr(SIZE);
    std::mt19937 gen(42);
    std::uniform_int_distribution<int> dist(1, 100);
    for (int& v : arr) v = dist(gen);

    auto t0 = std::chrono::high_resolution_clock::now();
    long long s1 = single_sum(arr);
    auto t1 = std::chrono::high_resolution_clock::now();
    double d1 = std::chrono::duration<double>(t1 - t0).count();

    t0 = std::chrono::high_resolution_clock::now();
    long long s2 = multi_sum(arr, 2);
    t1 = std::chrono::high_resolution_clock::now();
    double d2 = std::chrono::duration<double>(t1 - t0).count();

    t0 = std::chrono::high_resolution_clock::now();
    long long s4 = multi_sum(arr, 4);
    t1 = std::chrono::high_resolution_clock::now();
    double d4 = std::chrono::duration<double>(t1 - t0).count();

    std::cout << "1: sum=" << s1 << " time=" << d1 << "s\n";
    std::cout << "2: sum=" << s2 << " time=" << d2 << "s\n";
    std::cout << "4: sum=" << s4 << " time=" << d4 << "s\n";
    return 0;
}

Выводы

В ходе лабораторной работы было изучено влияние количества потоков на производительность вычислений при суммировании массива из 10 000 000 элементов.

Основные выводы:
  1. Многопоточность позволяет ускорить вычисления, если задачу можно разделить на независимые части.
  2. При использовании 2 и 4 потоков в C++ время выполнения уменьшается.
  3. Ускорение не линейное из-за накладных расходов на создание потоков.
  4. В Python модуль threading ограничен GIL. Для CPU-задач лучше использовать multiprocessing.
  5. Оптимальное число потоков обычно равно числу ядер процессора.
